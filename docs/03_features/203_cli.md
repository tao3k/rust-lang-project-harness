# CLI

The package includes a small `rust-project-harness` binary for local and CI
policy runs.

```shell
cargo run --bin rust-project-harness -- .
cargo run --bin rust-project-harness -- --json .
cargo run --bin rust-project-harness -- --agent-snapshot .
```

With no path argument, the CLI runs against the current working directory. With
one path argument, it runs the default project harness for that project root.

Compact text is the default output because it is the primary repair surface for
humans and LLM agents. The `--json` flag emits the serializable
`RustHarnessReport` shape for structured consumers.
The `--agent-snapshot` flag emits a parser reasoning-tree summary grouped by
package, source roots, owner branches, and finding groups. Use it when an agent
needs to orient itself before editing rather than inspect every discovered file.

Exit status follows the same policy as `assert_clean()`:

- `0`: no configured-blocking findings
- `1`: at least one configured-blocking finding
- `2`: CLI usage, project-root, or rendering error

`AGENT-*` findings remain visible in compact output by default but do not change
the exit status unless callers configure those severities as blocking through
the library API.
